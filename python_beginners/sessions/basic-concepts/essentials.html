Some essential concepts

The Essential Concepts

  • Before we can really get started, we need to know certain terms
  • We are going to cover them in enough detail to understand:
    • What the terms mean
    • How they relate to each other
    • How they fit into the overall scheme of a 'program'
  • We're going to give a few basic examples here, and then refer back to the theory when we're writing algorithms in our next session

The Essential Concepts

  • Literals
  • Identifiers and Variables
  • Data Types
  • Operators
  • Expressions
  • Functions

Literals

  • A literal is an actual piece of data that is written into a piece of code
  • Not taken from user input, retrieved from a database or generated from other values
  • E.g. having the number 2 written into your code is a literal
  • E.g. the text "hello" written straight into your code is a literal

Variables / Identifiers

  • When you're writing any sort of algorithm, you often want to refer back to previous values
  • When programming an algorithm, if you want to refer back to a previous calculation, you need to either recalculate it, or store it and retrieve it
  • You can use an identifier to label that piece of information for later use
  • When you label an ordinary data, that label is called a Variable
  • Identifiers in python are just words with a few rules about what you can call things
    • We'll cover the rules later
Variables and identifiers won't make much sense on their own, so we'll save the example for later....

Data Types

All applications, in one form or another, are about processing data
  • From a business application like a payroll, CRM, etc
  • To a computer game
  • Data comes in various types
  • What type the data is defines what we can do with it
  • Main data types are
    • Numbers
    • Strings (text)
    • Sequences (covered later)
  • There are lots of little details about how data types behave and are used, but they relate to other concepts, so they can be covered bit by bit

Operators

  • So we have some variables containing some data, of a particular data type
  • We can use operators to do something with them
  • Operators perform an action on or with something else
  • A good example is normal arithmetic operators that we can use in simple maths
    • +
    • -
    • *
    • /

Operators

  • The most important other operator at the moment is the assignment operator, which allows you to assign values to variables.
  • Assignment operators always need to be used with a variable - the thing the result or value is being assigned to
  • The variable being assigned to is always on the left of the expression (see next slide)
    • length = 5
    • total = 1 + 1
    • subtotal = previous_total + 1

Expressions

  • Expressions are the programming equivalent of sentences
  • They're made up of different combinations of the basic building blocks of programming
  • When you start putting together elements like variables, operators and literals, statements are formed
  • Expressions normally have a result

Functions

  • Units of code that carry out a task
  • Can take input
    • parameters (or arguments)
  • Which is processed in some way
  • And give back output
    • return values
  • This is very useful for a piece of code that you want to use a lot, especially when it is used in many different places
  • Python has lots of useful functions included in the language, that you can make immediate use of
  • Examples of function calls
    • result = function_name(input)
    • total = sum(1,2,3)
    • display(some_text)
