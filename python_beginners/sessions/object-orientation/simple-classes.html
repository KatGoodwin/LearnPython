Simple Classes in Python

What are Classes?

  • Like OO generally, classes are templates for object instances that are going to be created
  • Classes contain the definition of the data that can be stored and the actions that can be performed on the object
  • They can contain variables that belong to either the class or the instance
  • They can contain methods that can belong to either the class or the instance

Defining Classes

There are two styles of class with Python
  • 'Classic' Classes before python 2.2
    • These are classes without a built-in object in their base(s)
      • class ClassName:
      class ClassName( classic_class ):
  • New style Classes
    • These have a built-in object in their base(s):
      • class ClassName( object ):
      class ClassName( new_style_class ):
      class ClassName( int ):
To inherit from other classes in either version: 
    class SubClass(SuperClass1, SuperClass2)
  • Inheriting from a new-style class produces a new-style class.

Class and Instance Variables and Methods

  • Class methods can be accessed without creating an instance of the class
  • Instance methods are passed 'self' as the first parameter by Python, automatically.
  • They are invoked by calling them 'on' an instance of the class - eg instance.method()
  • Instance methods and attributes are specific to a particular instance of a class.
    • They cannot be accessed/used unless an instance exists.
    • They contain and use information that is specific to that instance
    • Instance variables are defined using the self namespace - eg self.variable

Methods and Access Specifiers


    Methods in objects are simply functions inside the object
  • Methods in Python do not have access specifiers (eg public, protected) like Java.
  • All methods are publically available
  • There is a convention in Python to indicate how functions should be treated
    • methods that are surrounded by single underscores are suggested as private to the class
    • methods that are surrounded by double underscores have special meanings in Python
    • You should feel guilty about using methods with an underscore.
It feels cleaner as an ex-Java programmer to call functions inside classes methods, and outside one functions...

Inheritance - Methods and Attributes


  • When one class inherits from another, the methods of the superclass (parent) become available to the subclass (child).
  • A class can be defined to inherit from multiple ('mixin') classes.
    • If methods are duplicated in multiple parents, there is a defined 'method resolution order'.
  • The superclass' methods can be over-ridden, either in the class definition, or at runtime.
