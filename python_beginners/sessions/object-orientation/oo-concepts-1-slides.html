Object Orientation Concepts - The Basics

Object Orientation - What and Why?


  • Intended to be a very broad introduction to / recap of OO Principles
  • Ignore programming language implementations completely initially
  • Object Orientation is a way of modelling the world
  • Objects should represent real world things

What is an Object?

  • Something that exists as a concept in the real world
  • Something you can point at and say "This is an xyz"
  • Something that has properties and a behaviour
  • Can be at various levels of granularity
    • person, user or administrator
    • animal, mammal or dog
    • kitchen_equipment, oven, fan assisted oven

Object Properties

  • Properties are aspects of an object that describe it
  • The properties of a generic person could be things like:
    • Hair colour
    • Age
    • Height
    • National Insurance number
    • Name

Object Actions

  • Actions are things that the object would perform or carry out

Objects, Classes and Instances

  • A Class is a template for creating lots of objects (instances) of.
  • An instance of a class is an object that has been created, and set up (normally with some values that are specific to it)
  • Each instance has its own values for properties, but the actions are common to that class

Inheritance

  • Often, classes will have some properties and actions in common
  • Rather than defining shared properties / actions for multiple objects, inheritance can be used to share behabviour
  • Unlike in nature, in OO, when one class inherits from another, it inherits all the behaviour of its 'parent'
  • It can then add to, or override the existing behaviour
  • Or add some completely new properties and behaviour
